Technical Specification

Protocol Documentation

JantaReport Digital Civic Infrastructure v2.4

1. System Architecture

JantaReport operates on a three-tier architecture:

  • Citizen Layer: Mobile-first web application for issue reporting
  • Authority Layer: Real-time dashboard for municipal agencies
  • Intelligence Layer: AI-powered categorization and routing system

2. Issue Lifecycle Protocol

Phase 1: Submission

  • Citizen submits issue with multimedia evidence
  • GPS coordinates auto-captured for precise location
  • AI categorizes issue type and severity
  • Unique tracking ID generated

Phase 2: Routing

  • Issue routed to appropriate municipal department
  • Real-time notification sent to city admin
  • Status updated to "SUBMITTED"

Phase 3: Resolution

  • Authority marks issue as "IN_PROGRESS"
  • Citizen receives real-time status updates
  • Upon completion, status changed to "RESOLVED"
  • Citizen can verify and provide feedback

3. Data Transmission Protocol

  • Transport: HTTPS with TLS 1.3 encryption
  • Authentication: JWT-based token system
  • Real-time Sync: WebSocket (Socket.io) for live updates
  • File Upload: Multipart form-data with 100MB limit
  • API Rate Limiting: 100 requests per minute per user

4. Role-Based Access Control

Citizen

  • • Submit issues
  • • Track status
  • • View history

City Admin

  • • View city issues
  • • Update status
  • • Assign tasks

Super Admin

  • • Global oversight
  • • Create admins
  • • Analytics access

5. AI Intelligence Protocol

Our AI system uses:

  • Emotion Detection: Keyword-based sentiment analysis (ANGRY, WORRIED, HELPFUL, NEUTRAL)
  • Auto-Categorization: NLP-based classification into 5 categories
  • Severity Prediction: Context-aware urgency assessment
  • Response Optimization: Empathetic, context-aware replies
